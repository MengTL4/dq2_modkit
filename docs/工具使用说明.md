# 工具使用说明

本文说明当前这套本地工具如何使用，覆盖三块内容：

- 运行时修改器：GUI 修改器和 CLI 命令。
- 离线存档编辑/存档编辑器流程：解密存档、编辑 JSON、重新加密成 `.rpgsave`。
- 解密脚本：导出 `data.pak`、`useData`、存档内容。

这些工具只放在 `dq2_modkit` 目录，不修改原游戏的 `package.json`、`www/index.html`、`www/guard.js`。

## 目录速览

```text
dq2_modkit/tools/                       工具脚本
dq2_modkit/tools/launch-gui.ps1         启动 GUI 修改器
dq2_modkit/tools/launch-save-editor.ps1 启动离线存档树形编辑器
dq2_modkit/tools/setup-runtime.ps1       从配置的游戏目录生成/刷新 NW 运行时链接
dq2_modkit/tools/clean-runtime.ps1       清理生成的 NW 运行时链接和字节码
dq2_modkit/tools/trainer-send.mjs        CLI 发送运行时命令
dq2_modkit/tools/extract-data-pak.mjs    解密导出 www/data.pak
dq2_modkit/tools/extract-usedata.mjs     解密导出 www/useData/*.data
dq2_modkit/tools/extract-saves.ps1       解密导出 www/save/*.rpgsave
dq2_modkit/tools/encrypt-saves.ps1       将存档 JSON 重新加密为 .rpgsave
dq2_modkit/config.example.json           游戏根目录本地配置示例
dq2_modkit/config.local.json             每个用户自己的游戏路径配置，不提交
dq2_modkit/skills/                       从零复刻本项目的 Codex skill

dq2_modkit/app/gui/                  GUI 修改器运行目录，app.ts 编译为 app.js
dq2_modkit/app/save-editor/          纯网页离线存档树形编辑器
dq2_modkit/runtime/trainer/              bridge 版游戏启动器运行目录
dq2_modkit/runtime/bridge/           注入游戏页面的 bridge 脚本
dq2_modkit/runtime/bridge-state/               bridge 状态、命令队列和事件日志

dq2_modkit/output/extract/data/                 data.pak 导出的 RPG Maker 数据
dq2_modkit/output/extract/useData/              useData 导出的 JSON/MessagePack
dq2_modkit/output/extract/save/                 解密后的存档 JSON/MessagePack
dq2_modkit/output/repack/save/                  重新加密后的存档输出
dq2_modkit/output/backup/save/                  GUI 备份存档的位置
```

`app/gui`、`runtime/trainer`、`runtime/save-harness` 里的 NW 运行时文件是生成产物，不是需要手动维护的源码。`app/save-editor` 是纯网页工具，不复制 `Game.exe`、`nw.dll` 这些运行时文件。启动脚本会在缺失时自动生成 NW 运行时；游戏更新后建议手动刷新一次：

```powershell
cd "<你的 dq2_modkit 目录>"
.\tools\setup-runtime.ps1 -Force
```

这个脚本会从配置的游戏根目录重新生成硬链接/目录联接，安装缺失的 Node 依赖，并从当前游戏版本的 `www/js/*.jsc.pak` 重新提取存档解密 harness 需要的字节码。

安装 Node 依赖时默认使用：

```text
https://registry.npmmirror.com
```

也可以手动指定 registry：

```powershell
.\tools\setup-runtime.ps1 -NpmRegistry "https://registry.npmmirror.com"
$env:DQ2_NPM_REGISTRY = "https://registry.npmmirror.com"
```

`launch-gui.ps1`、`launch-runtime.ps1`、`extract-saves.ps1`、`extract-all.ps1` 在自动触发 setup 时也支持同一个 `-NpmRegistry` 参数。`launch-save-editor.ps1` 也支持 `-NpmRegistry`，但它只安装网页编辑器依赖，不会触发 NW 运行时生成。

GUI 修改器源码使用 TypeScript。日常使用直接运行 `launch-gui.ps1` 即可；如果你修改了 `app/gui/app.ts`，启动脚本会在发现 `app.ts` 比 `app.js` 新时自动安装 GUI 构建依赖并编译。也可以手动构建：

```powershell
cd "<你的 dq2_modkit 目录>\app\gui"
npm.cmd install --registry https://registry.npmmirror.com
npm.cmd run build
```

如果想清空生成的运行时产物：

```powershell
cd "<你的 dq2_modkit 目录>"
.\tools\clean-runtime.ps1
```

默认只删除 NW 运行时硬链接/目录联接、存档 harness 字节码和探针结果，不删除 `node_modules`。需要连依赖目录一起清理时：

```powershell
.\tools\clean-runtime.ps1 -IncludeDependencies
```

清理后下次启动 GUI、启动 bridge 或解密存档时会自动重新生成；也可以手动运行 `.\tools\setup-runtime.ps1`。

## 复刻 Skill

`dq2_modkit/skills` 是随项目保存的 Codex skill，用来让另一个 Codex 从零复刻这套项目。它包含：

- `SKILL.md`：触发说明和核心流程。
- `references/rebuild-playbook.md`：从零实现路线。
- `references/formats-and-contracts.md`：数据格式、加解密、bridge 命令和 hook 合约。
- `assets/dq2_modkit_template`：不含运行时生成产物的干净模板。
- `scripts/scaffold-dq2-modkit.ps1`：把模板复制到目标游戏目录的脚本。

在当前游戏根目录下可以这样测试 scaffold：

```powershell
.\dq2_modkit\skills\scripts\scaffold-dq2-modkit.ps1 -GameRoot "." -DryRun
```

要对另一个游戏目录生成项目：

```powershell
& ".\dq2_modkit\skills\scripts\scaffold-dq2-modkit.ps1" `
  -GameRoot "目标游戏目录" `
  -RunSetup
```

如果需要让 Codex 自动发现这个 skill，可以把 `dq2_modkit/skills` 作为 `dq2-modkit-builder` 放到 Codex 的全局 skills 目录；随项目归档时保持当前路径即可。

## 游戏根目录配置

最推荐的布局是把 `dq2_modkit` 直接放在游戏根目录下，也就是和 `Game.exe`、`www` 同级。这种情况下什么都不用配，工具会自动把 `dq2_modkit` 的父目录当作游戏根目录。

只有当项目放在别的位置，或要让不同用户使用自己的 Steam 安装路径时，才需要配置游戏根目录。不要改脚本源码，进入项目目录后复制本地配置：

```powershell
cd "<你的 dq2_modkit 目录>"
Copy-Item .\config.example.json .\config.local.json
notepad .\config.local.json
```

把 `gameRoot` 改成包含 `Game.exe` 和 `www\index.html` 的游戏根目录。`config.local.json` 已写入 `.gitignore`，每个用户都可以保存自己的路径，不会污染仓库。

游戏根目录解析顺序是：

1. 脚本参数 `-GameRoot "..."`。
2. 环境变量 `DQ2_GAME_ROOT`。
3. `dq2_modkit/config.local.json` 里的 `gameRoot`。
4. 兼容旧布局：`dq2_modkit` 的父目录。

临时指定目录也可以直接传参数：

```powershell
.\tools\launch-gui.ps1 -GameRoot "<你的游戏根目录>"
.\tools\setup-runtime.ps1 -GameRoot "<你的游戏根目录>" -Force
.\tools\extract-all.ps1 -GameRoot "<你的游戏根目录>"
```

`-GameRoot` 可以写绝对路径，也可以写相对当前 PowerShell 所在目录的相对路径。

## 使用前准备

从项目目录打开 PowerShell：

```powershell
cd "<你的 dq2_modkit 目录>"
```

### 安装 Node.js

工具脚本需要系统命令行里的 `node` 和 `npm`，不是游戏目录里的 `node.dll`。先检查：

```powershell
node --version
npm.cmd --version
```

脚本最低要求 Node.js 18+；新装建议直接安装当前 LTS 版。可以使用 Windows 包管理器：

```powershell
winget install -e --id OpenJS.NodeJS.LTS
```

也可以从 [Node.js 官网](https://nodejs.org/zh-cn/download) 下载 Windows LTS 安装包。安装后重新打开 PowerShell，再确认 `node --version` 和 `npm.cmd --version` 可用。

在 Windows PowerShell 里，`npm` 可能优先命中 `npm.ps1`，如果执行策略拦截它，直接使用 `npm.cmd` 即可。本项目的 setup 脚本也会优先调用 `npm.cmd`。

### PowerShell 执行策略

如果运行 `.ps1` 时出现“在此系统上禁止运行脚本”之类的提示，有两种方式。

推荐只对当前用户放开本地脚本：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

不想修改执行策略时，可以每次用 `Bypass` 单次运行：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\tools\launch-gui.ps1"
```

其它 `.ps1` 同理，例如：

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File ".\tools\setup-runtime.ps1" -Force
```

推荐习惯：

- 使用运行时修改器前，先关闭旧的 bridge 版游戏窗口，避免旧版 bridge 没加载新脚本。
- 游戏更新后，先运行 `tools/setup-runtime.ps1 -Force` 刷新 NW 运行时链接和存档 harness 字节码。
- 清理运行时产物前，先关闭 GUI 修改器、bridge 版游戏和存档解密 harness。
- 做离线存档编辑前，先关闭游戏。
- 替换 `www/save` 里的 `.rpgsave` 前，先备份整个 `www/save`。
- 不要手动编辑原游戏的 `package.json`、`www/index.html`、`www/guard.js`。

## GUI 修改器

启动 GUI：

```powershell
cd "<你的 dq2_modkit 目录>"
.\tools\launch-gui.ps1
```

GUI 打开后：

1. 点击右上角“启动游戏”。
2. 等状态从“加载中”变成“已连接”。
3. 再使用金币、物品、角色、地图、杂项等功能。

如果右上角显示“需重启”，说明当前游戏窗口里的 bridge 版本旧了。关闭 bridge 版游戏窗口，再从 GUI 重新点击“启动游戏”。

### 常用

常用页包含：

- 金币设定、金币增加、金币预设。
- 经验倍率、金币倍率、掉率倍率、技能熟练度倍率。
- 倍率预设：`5x`、`10x`、`20x`、`50x`、`100x`、重置。
- 技能无耗：开启后会守住队伍成员的 MP/TP，不让技能扣低。
- 队伍恢复：恢复当前队伍成员。
- 保存：调用游戏自己的保存函数保存到指定槽位。
- 刷新标题：刷新标题界面的继续/加载状态。

倍率和技能无耗只影响当前 bridge 版游戏进程。关闭游戏后会失效，下次启动需要重新设置。

### 物品角色

物品面板：

- 类型可选：全部、物品、武器、防具。
- “全部”会把物品、武器、防具三类合并到同一个列表。
- 搜索支持 ID、名称、描述。
- 手动输入 ID 时：
  - 单类模式可以直接输入数字，例如 `5`。
  - 全部模式可以输入 `item:5`、`weapon:2`、`armor:3`。
- 列表里每一行可以直接点击“添加”。

角色编辑：

- 搜索角色名称或 ID。
- 解锁人物/加入队伍。
- 离队。
- 恢复。
- 设定等级。
- 增加经验。
- 写入 HP / MP / TP。
- 增加参数：最大 HP、最大 MP、攻击、防御、魔攻、魔防、敏捷、幸运。

技能面板：

- 选择角色 ID。
- 搜索技能名称、ID 或描述。
- 对指定角色执行“学会”或“遗忘”。

### 地图事件

地图传送：

- 搜索地图名称或 ID。
- 输入地图 ID、X、Y、朝向、淡入方式。
- 点击“传送”。
- 可以记录当前位置，并返回记录点。

公共事件：

- 搜索公共事件名称或 ID。
- 输入事件 ID。
- 点击“运行事件”。

公共事件是游戏内部事件，运行前最好先保存。

### 杂项

杂项页包含：

- 变量浏览、搜索、写入。
- 开关浏览、搜索、写入。
- 解锁敌人图鉴。

变量和开关来自 `dq2_modkit/output/extract/data/System.json`。写入值时，GUI 会尽量把 `true`、`false`、数字、`null` 和 JSON 文本解析成对应类型。

解锁敌人图鉴会写入游戏配置对象并调用配置保存。执行后需要重新打开游戏里的图鉴菜单查看。

### 调试

调试页可以发送自定义 JSON 命令。例如：

```json
{ "type": "ping" }
```

常用时建议优先使用 GUI 按钮；自定义命令适合测试 bridge 是否在线，或发送 GUI 暂时没有做按钮的命令。

## 离线存档编辑 / 存档编辑器流程

现在有两种离线方式：

- 可视化树形编辑器：直接打开 `.rpgsave`，编辑 JSON 树，导出新的 `.rpgsave`。
- 脚本流程：批量解密到 `output/extract/save`，手动编辑 JSON，再批量重新加密。

两种方式都不启动游戏、不注入运行时、不修改原游戏文件。做离线存档编辑前仍然建议先关闭游戏。

### 可视化树形编辑器

启动：

```powershell
cd "<你的 dq2_modkit 目录>"
.\tools\launch-save-editor.ps1
```

常用参数：

```powershell
.\tools\launch-save-editor.ps1 -Port 5174
.\tools\launch-save-editor.ps1 -NoOpen
.\tools\launch-save-editor.ps1 -NpmRegistry "https://registry.npmmirror.com"
```

这个脚本会启动本地 Vite 页面，默认地址：

```text
http://127.0.0.1:5174
```

第一次启动会在 `app/save-editor` 下安装网页依赖。这个模块只处理本地文件，不需要 `Game.exe`、`nw.dll` 或 bridge。

编辑流程：

1. 点击“打开存档”，选择 `config.rpgsave`、`global.rpgsave` 或 `fileN.rpgsave`。
2. 在 JSON 树里修改节点。
3. 点击“导出 JSON”可导出中间 JSON。
4. 点击“导出存档”生成新的 `.rpgsave`。
5. 备份原存档后，再把导出的 `.rpgsave` 放回游戏 `www/save` 测试。

槽位 ID 会从文件名推断：`global.rpgsave` 是 `0`，`file1.rpgsave` 是 `1`，`file3.rpgsave` 是 `3`。如果文件名不标准，需要手动填写槽位 ID。`config.rpgsave` 不需要槽位 ID。

这个编辑器只做原始 JSON 树编辑，不提供金币、物品、角色之类的便捷按钮。

### 脚本流程

脚本流程的思路是：

```text
www/save/*.rpgsave
  -> 解密导出为 JSON
  -> 手动编辑 JSON
  -> 重新加密为 .rpgsave
  -> 放回 www/save 测试
```

#### 1. 解密导出存档

关闭游戏后执行：

```powershell
cd "<你的 dq2_modkit 目录>"
.\tools\extract-saves.ps1
```

输出目录：

```text
dq2_modkit/output/extract/save
```

常见输出：

```text
config.json
config.msgpack
global.json
global.msgpack
file1.json
file1.msgpack
file3.json
file3.msgpack
```

当前解密探针默认处理 `global`、`file1`、`file3` 和 `config`。如果以后要处理其它槽位，需要扩展 `dq2_modkit/runtime/save-harness/index.html` 里的存档 ID 列表。

#### 2. 编辑 JSON

推荐优先编辑 `.json` 文件，不要直接编辑 `.msgpack`。

常见文件：

- `config.json`：游戏配置、敌人图鉴、标题/全局设置等。
- `global.json`：全局存档索引信息。
- `file1.json`、`file3.json`：具体存档槽内容。

常见字段：

- `fileN.json -> party._gold`：金币。
- `fileN.json -> party._items`：物品数量，key 是物品 ID。
- `fileN.json -> party._weapons`：武器数量，key 是武器 ID。
- `fileN.json -> party._armors`：防具数量，key 是防具 ID。
- `fileN.json -> party._actors`：队伍角色列表。
- `fileN.json -> actors._data`：角色实例数据。
- `fileN.json -> switches`：开关数据。
- `fileN.json -> variables`：变量数据。
- `fileN.json -> system._revealedEnemyWeaknesses`：敌人弱点/图鉴相关记录。
- `config.json -> enemyBook`：敌人图鉴配置记录。

注意事项：

- JSON 里出现的 `@c`、`@a`、`@` 是游戏序列化器留下的引用/类型辅助字段，不要随意删除。
- 物品数量里可能存在小数形式，这是游戏自己的数据状态；如果只是想加物品，优先用运行时修改器。
- 修改角色结构时不要删除角色对象里的未知字段。
- 大改之前保留一份原 JSON，方便回退。

#### 3. 重新加密

默认把 `dq2_modkit/output/extract/save` 重新打包到 `dq2_modkit/output/repack/save`：

```powershell
cd "<你的 dq2_modkit 目录>"
.\tools\encrypt-saves.ps1
```

指定输入输出目录：

```powershell
.\tools\encrypt-saves.ps1 -InputDir ".\output\extract\save" -OutputDir ".\output\repack\save"
```

指定要打包的存档 ID：

```powershell
.\tools\encrypt-saves.ps1 -Ids @(0,1,3)
```

不打包 `config.rpgsave`：

```powershell
.\tools\encrypt-saves.ps1 -NoConfig
```

输出目录会生成：

```text
config.rpgsave
global.rpgsave
file1.rpgsave
file3.rpgsave
_repack-report.json
```

`_repack-report.json` 里每个条目的 `verified` 应该是 `true`。脚本会把新生成的 `.rpgsave` 再解回来和输入 JSON 比对，只有比对通过才算可靠。

#### 4. 放回游戏测试

替换存档前先备份：

```powershell
$GameRoot = "<你的游戏根目录>"
Copy-Item -LiteralPath (Join-Path $GameRoot "www\save") -Destination ".\output\backup\save\manual-backup" -Recurse -Force
```

然后把 `output/repack/save` 里需要测试的 `.rpgsave` 放回游戏目录的 `www/save` 对应文件名。测试时建议一次只替换一个槽位，例如只替换 `file1.rpgsave`。

## 推荐工作流

只想改当前游戏进程：

```text
启动 GUI -> 启动游戏 -> 等待已连接 -> 使用 GUI 功能 -> 游戏内保存
```

想离线改存档：

```text
关闭游戏 -> extract-saves -> 编辑 JSON -> encrypt-saves -> 备份原存档 -> 替换测试
```

想查 ID：

```text

```

想查 useData：

```text
extract-usedata -> 查看 dq2_modkit/output/extract/useData 下的 JSON 和 _index.json
```

## 常见问题

### GUI 显示“未连接”

说明 bridge 版游戏没启动，或游戏还没初始化到队伍对象可用。点击 GUI 的“启动游戏”，等状态变成“已连接”。

### GUI 显示“需重启”

说明游戏窗口加载的是旧 bridge。关闭 bridge 版游戏窗口，再从 GUI 启动。

### 命令发了没效果

先看：

```powershell
node .\tools\trainer-send.mjs status
```

确认：

- `hasParty=true`
- `bridgeVersion` 和 GUI 期望一致
- `lastError` 为空

再看：

```text
dq2_modkit/runtime/bridge-state/events.jsonl
```

如果事件里是 `FAIL`，里面会有错误原因。

### 重新加密后的存档大小和原文件不一样

正常。新文件会使用随机 IV，zlib 压缩结果也可能和游戏原始保存不同。以 `_repack-report.json` 的 `verified=true` 为准。

### 能不能把 data.pak 或 useData 改回游戏里

当前脚本实现的是导出查看。存档已经实现重新加密；`data.pak` 和 `useData` 的重新封包/回写没有作为常规工具暴露。
